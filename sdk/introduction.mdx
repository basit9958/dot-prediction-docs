The Dot Prediction TypeScript SDK provides a type-safe client for querying prediction markets across **Polymarket**, **Kalshi**, and **Limitless** with a single interface.

## Installation

<CodeGroup>

```bash npm
npm install @dot-prediction/sdk
```

```bash yarn
yarn add @dot-prediction/sdk
```

```bash pnpm
pnpm add @dot-prediction/sdk
```

</CodeGroup>

## Quick start

```typescript
import { DotPredictionClient } from '@dot-prediction/sdk';

const client = new DotPredictionClient({
  apiKey: 'your-api-key',
});

// List markets from Polymarket
const { markets, pagination } = await client.markets.list({
  provider: 'polymarket',
  limit: '10',
  status: 'active',
});

// Get a specific market
const market = await client.markets.getById({
  provider: 'kalshi',
  id: 'KXBTC-25FEB28',
});

// Check API health
const health = await client.system.health();
console.log(health.status); // "healthy"
```

## Configuration

```typescript
const client = new DotPredictionClient({
  // Required: Your API key
  apiKey: 'your-api-key',

  // Optional: Custom base URL (defaults to https://api.dotprediction.com/v1/api)
  baseURL: 'https://api.dotprediction.com/v1/api',
});
```

The SDK authenticates via the `X-API-Key` header. Obtain your API key from the [quickstart guide](/quickstart).

## Providers

Most endpoints require a `provider` parameter. Supported providers and their capabilities:

| Provider | Markets | Events | Series | Orderbook | Trades | Price History | Profile |
|----------|---------|--------|--------|-----------|--------|---------------|---------|
| `polymarket` | Yes | Yes | Yes | Yes | Yes | Yes | Yes |
| `kalshi` | Yes | Yes | Yes | Yes | Yes | Yes | Yes |
| `limitless` | Yes | No | No | Yes | No | Yes | No |

## Available modules

The client exposes five modules:

| Module | Description |
|--------|-------------|
| `client.markets` | List markets, get details, orderbooks, trades, and price history |
| `client.events` | List and get events (groups of related markets) |
| `client.series` | List and get series (groups of related events) |
| `client.profile` | User profiles, trades, PnL, positions, and activity |
| `client.system` | Health checks and platform capabilities |
